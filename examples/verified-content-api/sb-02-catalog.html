1필터 교육과정 필터#filters

학년·과목·단원·성취기준으로 결과를 좁힌다. 교육과정 정렬 자산이 곧 차별화.

이벤트 선택 변경 → 결과 즉시 필터 + URL 쿼리·API 미리보기 동기화 데이터 교육과정 메타 {grade, subject, unit, achievement_standard_code} 예외 결과 0건 → 빈 상태 + 인접 성취기준 추천
2검색 키워드 검색#q

문항/해설 본문 검색.

이벤트 입력 → 300ms 디바운스 → 검색 데이터 q: full-text(문항·해설), 필터와 AND 결합 예외 빈 검색=전체 / 특수문자 escape
3결과 문항 카드.item-card

검증 문항 1건. 정답·해설은 토글로 노출.

이벤트 "정답/해설 보기" 토글 → 펼침 데이터 item {stem, choices, answer, explanation, curriculum_code, source} 상태 미인증/무료 티어 → 정답·해설 블러 + 키 발급 유도 예외 라이선스 범위 외 항목 → 잠금 + 플랜 안내
4신뢰 검증 콘텐츠 배지.badge-verified

항목 단위 신뢰 표식. 검수 메타로 보증.

데이터 verified=true + {검수자, 검수일, 출처} 예외 미검증 항목엔 배지 미표시(검증 자산만 신뢰 레이어)
5API 엔드포인트 미리보기#api-preview

현재 필터를 실제 호출 코드로. "탐색=통합" 전환 다리.

이벤트 📋 클릭 → 코드 복사 / 필터 변경 시 자동 갱신 데이터 GET /v1/items?grade=&subject=&unit=&standard= (cURL/JS 탭) 예외 키 없음 → 401 예시 + "무료 키 받기" 링크
6푸터 건수·쿼터·페이지#pager

결과 규모와 잔여 호출 쿼터를 함께 노출.

이벤트 다음/이전 또는 무한 스크롤 데이터 {total, page, remaining_quota} 상태 쿼터 소진 임박(예: 90%) 경고 배너
관련 상세 명세: /v1/items 응답 스키마, 라이선스 티어별 노출 정책, 검증 메타 데이터 모델(다음 단계)
